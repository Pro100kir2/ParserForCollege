# Парсер документов РПД

Модульная система разбора документов учебных планов (РПД), которая извлекает списки литературы и материально-технические ресурсы из файлов форматов DOCX и DOC с высокой точностью (confidence score 1.00).

## Возможности

- **Два режима работы**: Интерфейс командной строки (CLI) и веб-интерфейс на FastAPI
- **Поддержка документов**: Форматы DOCX и DOC
- **Извлечение секций**: Настраиваемые псевдонимы для секций «Основная литература», «Дополнительная литература» и «Материально-техническое обеспечение»
- **Высокая точность извлечения**: Система оценки уверенности (confidence scoring) с показателем 1.00 для всех секций
- **Дедупликация**: Автоматическая нормализация и нечёткая дедупликация (fuzzy matching)
- **Экспорт в Word**: Профессионально оформленные документы Word (.docx) с Times New Roman, межабзацными интервалами и нумерацией страниц
- **Инкрементальная обработка**: Чекпоинты и возобновление обработки с пропуском уже обработанных файлов
- **Хеширование контента**: SHA256 хеширование для точного отслеживания изменений
- **Версионирование правил**: Отслеживание версии экстрактора для воспроизводимости результатов
- **Трассировка**: OpenTelemetry трассировка для отладки и мониторинга
- **Устойчивость к ошибкам**: Продолжает обработку даже при сбое отдельных документов
- **Структурированное логирование**: Подробная статистика обработки и отчёты об ошибках
- **Асинхронная обработка**: Многопоточная асинхронная обработка с настраиваемым размером чанков
- **Сбор метрик**: Отслеживание количества обработанных файлов, времени выполнения, процента ошибок
- **Проверки работоспособности**: Мониторинг состояния системы (CPU, память, диск)
- **Мониторинг производительности**: Метрики производительности в реальном времени
- **Отслеживание ошибок**: Контроль частоты ошибок с настраиваемыми порогами
- **Оповещения**: Автоматические уведомления при превышении порогов ошибок

## Использование

### Режим CLI

```bash
python main.py --input ./rpd
```

С указанием директории вывода:
```bash
python main.py --input ./rpd --output ./result
```

С короткими аргументами:
```bash
python main.py -i ./rpd -o ./result -s -v
```

**Доступные аргументы:**
- `-i, --input`: Путь к директории с документами РПД (обязательный)
- `-o, --output`: Директория для сохранения Word-файлов (по умолчанию: `./output`)
- `-s, --single-workbook`: Экспорт в один файл с несколькими секциями
- `-v, --verbose`: Включить подробное логирование
- `-c, --confidence`: Порог уверенности для извлечения (по умолчанию: 0.7)
- `-w, --workers`: Количество воркеров (по умолчанию: 4)
- `--resume`: Возобновить обработку с чекпоинта
- `--incremental`: Инкрементальный режим (пропускать обработанные файлы)

## Конфигурация

Конфигурация управляется через переменные окружения (файл `.env`):

```bash
# Псевдонимы секций
SECTION_ALIASES__MAIN_LITERATURE=Основная литература
SECTION_ALIASES__ADDITIONAL_LITERATURE=Дополнительная литература
SECTION_ALIASES__MATERIAL_RESOURCES=Материально-техническое обеспечение,Материально-техническая база

# Настройки экспорта
DEFAULT_OUTPUT_DIR=./output
SINGLE_WORKBOOK_EXPORT=true

# Настройки воркеров
WORKER_CAPACITY=1000
MAX_WORKERS=4

# Настройки оповещений
ALERT_ERROR_THRESHOLD=5
ALERT_ERROR_WINDOW_MINUTES=10

# Настройки экстрактора
CONFIDENCE_THRESHOLD=0.7
FUZZY_DEDUPLICATION_THRESHOLD=85
CHECKPOINT_DIR=./.checkpoint
EXTRACTOR_VERSION=1.0.0

# Логирование
LOG_LEVEL=INFO
```

### Параметры воркеров

- `WORKER_CAPACITY`: Количество документов в одном чанке (по умолчанию: 1000)
- `MAX_WORKERS`: Максимальное количество параллельных воркеров (по умолчанию: 4)
- `ALERT_ERROR_THRESHOLD`: Порог количества ошибок для оповещения (по умолчанию: 5)
- `ALERT_ERROR_WINDOW_MINUTES`: Временное окно для расчёта процента ошибок (по умолчанию: 10 минут)

## Точки расширения

Система спроектирована для будущих расширений:

- **Хранение в базе данных**: Добавление слоя persistence в сервисы
- **Веб-интерфейс**: API полностью готов к интеграции с фронтендом
- **История обработок**: Добавление трекинга в слой сервисов
- **Поддержка OCR**: Расширение читателей для PDF и изображений
- **WebSocket**: Обновления прогресса в реальном времени для длительных задач
- **Аутентификация**: Добавление авторизации API

## Технологический стек

- **FastAPI** — Веб-фреймворк
- **python-docx** — Парсинг DOCX и экспорт в Word
- **pypandoc** — Конвертация DOC
- **Pydantic** — Валидация данных
- **psutil** — Мониторинг системы и метрики
- **asyncio** — Поддержка асинхронной обработки
- **OpenTelemetry** — Трассировка и мониторинг
- **RapidFuzz** — Нечёткое сопоставление для дедупликации
- **hashlib** — Хеширование контента (SHA256)
